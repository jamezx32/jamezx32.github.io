    V8 调试手册¶

    概述¶

    本文记录 V8 漏洞分析常用的源码获取、构建配置、d8 参数、gdb 辅助命令和分析流程。

    建议同时准备两个构建目录：

    构建目录 用途
    x64.debug 源码调试、断点、对象打印
    x64.release_with_symbol 漏洞复现、符号化分析

    安装 depot_tools¶

    git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git ~/depot_tools
echo 'export PATH="$HOME/depot_tools:$PATH"' >> ~/.bashrc
source ~/.bashrc
gclient

    depot_tools 建议放在 PATH 前部，避免系统自带 gn、ninja 影响构建。

    获取源码¶

    mkdir -p ~/v8
cd ~/v8
fetch v8
cd v8
gclient sync

    Linux 依赖：

    ./build/install-build-deps.sh

    切换到指定 tag 或 commit：

    cd ~/v8/v8
git fetch --tags
git checkout <tag-or-commit>
gclient sync -D

    代理配置¶

    网络受限环境可临时配置代理：

    proxy_on() {
  export http_proxy=http://127.0.0.1:xxx
  export https_proxy=$http_proxy
  git config --global http.proxy $http_proxy
  git config --global https.proxy $http_proxy
}

proxy_off() {
  unset http_proxy
  unset https_proxy
  git config --global --unset http.proxy
  git config --global --unset https.proxy
}

    Debug 构建¶

    tools/dev/v8gen.py x64.debug
ninja -C out/x64.debug d8

    Release With Symbol 构建¶

    gn gen out/x64.release_with_symbol --args='
is_debug=false
is_component_build=true
symbol_level=2
target_cpu="x64"
v8_enable_backtrace=true
v8_enable_disassembler=true
v8_enable_object_print=true
v8_enable_verify_heap=true
treat_warnings_as_errors=false
'
ninja -C out/x64.release_with_symbol d8

    验证版本：

    ./out/x64.release_with_symbol/d8 --version

    gdbinit_v8¶

    V8 仓库提供 tools/gdbinit，可向 gdb 注入 V8 专用命令。

    cp tools/gdbinit ~/.gdbinit_v8
echo 'source ~/.gdbinit_v8' >> ~/.gdbinit

    启动方式：

    gdb --args ./out/x64.debug/d8 poc.js --allow-natives-syntax

    常用命令：

    命令 说明
    job <tagged_ptr> 打印对象
    jlh <local_handle> 打印 local handle 指向对象
    jh <internal_handle> 打印 internal handle 指向对象
    jco [pc] 打印 PC 对应 Code 对象
    jca [pc] [n] 打印附近反汇编
    jst / jstc 打印 JS 栈
    jtt <map> 打印 map transition tree

    常用配合：

    1. 在 JS 中使用 %DebugPrint(obj) 输出对象地址
    2. 在 JS 中使用 %SystemBreak() 中断进程
    3. 在 gdb 中使用 job <addr> 查看对象

    常用启动参数¶

    ./out/x64.debug/d8 poc.js \
  --allow-natives-syntax \
  --print-bytecode \
  --print-bytecode-filter=trigger \
  --trace-opt \
  --trace-deopt \
  --trace-ic \
  >stdout.v8dump 2>stderr.v8dump
    参数 说明
    --allow-natives-syntax 启用 %DebugPrint、%OptimizeFunctionOnNextCall 等内建
    --print-bytecode 输出 Ignition bytecode
    --print-bytecode-filter=<name> 过滤指定函数 bytecode
    --trace-opt 输出优化编译日志
    --trace-deopt 输出 deoptimization 日志
    --trace-ic 输出 IC 状态迁移

    命令模板¶

    输出 bytecode：

    ./out/x64.debug/d8 poc.js \
  --print-bytecode \
  --print-bytecode-filter=trigger

    输出优化和 deoptimization：

    ./out/x64.debug/d8 poc.js \
  --allow-natives-syntax \
  --trace-opt \
  --trace-deopt

    输出 TurboFan 图：

    mkdir -p turbo
./out/x64.debug/d8 poc.js \
  --trace-turbo \
  --trace-turbo-path=./turbo \
  --trace-turbo-filter=trigger

    提升复现稳定性：

    ./out/x64.debug/d8 poc.js \
  --predictable \
  --random-seed=1337

    参数名称可能随 V8 版本变化，使用前应通过以下命令校对：

    ./out/x64.debug/d8 --help

    GDB 常用操作¶

    gdb --args ./out/x64.debug/d8 poc.js --allow-natives-syntax

    (gdb) set pagination off
(gdb) run
(gdb) bt
(gdb) info registers
(gdb) x/20gx $rsp

    常见断点：

    (gdb) rbreak Builtins_
(gdb) b v8::internal::Runtime_ThrowReferenceError
(gdb) b v8::internal::BytecodeGenerator::ControlScopeForBreakable::Execute

    % 内建¶

    %DebugPrint(obj);
%SystemBreak();
%PrepareFunctionForOptimization(f);
%OptimizeFunctionOnNextCall(f);

    类型混淆分析中，可结合 x/gx 核对语义字段和内存偏移。

    漏洞分析流程¶

    1. 最小化 PoC，保留稳定触发条件
    2. 输出 bytecode，确认前端语义和控制流
    3. 对比未优化与已优化行为，判断是否依赖 JIT
    4. 打印关键对象，包括 receiver、lookup object、返回值和数组 backing store
    5. 输出优化日志、deopt 日志、IC 日志和 TurboFan 图
    6. 对照 patch，确认修复点与本地断点是否位于同一路径

    速查¶

    # gdbinit_v8
cp tools/gdbinit ~/.gdbinit_v8
echo 'source ~/.gdbinit_v8' >> ~/.gdbinit

# bytecode
./out/x64.debug/d8 poc.js --print-bytecode --print-bytecode-filter=trigger

# opt / deopt
./out/x64.debug/d8 poc.js --allow-natives-syntax --trace-opt --trace-deopt

# IC
./out/x64.debug/d8 poc.js --trace-ic

# TurboFan
./out/x64.debug/d8 poc.js --trace-turbo --trace-turbo-path=./turbo --trace-turbo-filter=trigger

# gdb
gdb --args ./out/x64.debug/d8 poc.js --allow-natives-syntax
