# V8 数组的内存布局

## 概述

JavaScript 数组在 V8 中仍是对象。命名属性和整数索引分别存储在不同路径中：

```javascript
arr.a = 1;   // properties
arr[0] = 1; // elements
```

本文中的布局图为调试视角下的逻辑结构，不代表所有 V8 版本的精确字段偏移。

## JSArray 结构

普通 `JSArray` 的核心字段包括：

| 字段 | 说明 |
|---|---|
| `map` | 描述对象类型、隐藏类和 elements kind |
| `properties` | 命名属性存储 |
| `elements` | 整数索引元素的 backing store |
| `length` | JavaScript 层可见数组长度 |

`map` 决定 `elements` 的解释方式。分析数组对象时，应先确认 `map` 和 elements kind。

## Elements Kind

常见 elements kind 包括：

| elements kind | backing store | 槽大小 | 内容 |
|---|---|---|---|
| `PACKED_SMI_ELEMENTS` / `HOLEY_SMI_ELEMENTS` | `FixedArray` | 4 字节 | `Smi` |
| `PACKED_DOUBLE_ELEMENTS` / `HOLEY_DOUBLE_ELEMENTS` | `FixedDoubleArray` | 8 字节 | `Float64` |
| `PACKED_ELEMENTS` / `HOLEY_ELEMENTS` | `FixedArray` | 4 字节 | tagged value / 对象引用 |

`PACKED_*` 表示密集数组，`HOLEY_*` 表示允许数组空洞。

## FixedDoubleArray

纯浮点数组通常使用 `PACKED_DOUBLE_ELEMENTS`，其 `elements` 指向 `FixedDoubleArray`。元素以连续 8 字节 double 存储。

```javascript
let arr = [1.1, 2.2, 3.3];
```

## FixedArray

对象数组、混合类型数组和泛化后的数组通常使用 `FixedArray`。槽位保存 tagged value。64 位并启用 pointer compression 时，槽位通常按 4 字节压缩表示观察。

![V8 object array layout](images/对象型数组.png)

`PACKED_SMI_ELEMENTS` 也使用 `FixedArray`，但槽位保存的是 `Smi`。

## Elements Kind 转换

elements kind 可随写入内容变化，并通常向更通用的方向迁移：

```text
PACKED_SMI_ELEMENTS
  -> PACKED_DOUBLE_ELEMENTS
  -> PACKED_ELEMENTS
```

示例：

- 全部元素为小整数时使用 `PACKED_SMI_ELEMENTS`
- 写入浮点数后可迁移到 `PACKED_DOUBLE_ELEMENTS`
- 写入对象或字符串后可迁移到 `PACKED_ELEMENTS`

空洞会触发 `PACKED_* -> HOLEY_*` 转换。该转换改变密集/稀疏语义，通常不改变 backing store 大类：double elements 仍使用 `FixedDoubleArray`，其余通常使用 `FixedArray`。

## 分析检查项

- `map` 是否匹配预期 elements kind
- `elements` 是否指向 `FixedArray` 或 `FixedDoubleArray`
- 槽位应按 tagged value 还是 raw double 解释
- 数组是否已经从 `PACKED_*` 迁移到 `HOLEY_*`
- `length` 与 backing store capacity 是否保持一致
