# V8 指针压缩机制

## 概述

64 位 V8 中，许多 JavaScript 值以 tagged value 形式存储。若所有 tagged value 均按 64 位保存，对象字段、元素槽和属性槽会产生较高内存开销。Pointer Compression 用于降低 tagged value 的存储成本。

Pointer Compression 压缩的是内存中的存储表示。运行时参与解引用或地址计算时，压缩值会恢复为可使用的 64 位形式。

## 地址窗口

Pointer Compression 依赖一个 4GB 地址窗口。相关堆对象可通过 32 位偏移描述，并在使用时结合 isolate root / cage base 恢复为完整地址。

该机制不表示 V8 进程只能使用 4GB 内存，也不表示所有 C++ 指针都变成 32 位。

## Tagged Value

V8 常见 tagged value 包括：

| 类型 | 说明 |
|---|---|
| `Smi` | 直接编码在 tagged value 中的小整数 |
| `HeapObject` | 指向堆对象的引用，压缩后保存为可恢复地址表示 |

调试 tagged value 时，最低位通常用于区分 `Smi` 与对象引用。该规则适用于 tagged value 语义，不适用于任意 C++ 指针字段。

## 压缩形态

```text
|---------- 32 bits ----------|
Pointer: |____30 位有效载荷____w1|
Smi:     |_____31 位整数值______0|
```

`w` 表示 strong / weak 等附加标记位。对象引用保存压缩表示，`Smi` 保存内联整数编码。

## 常见误区

| 误区 | 说明 |
|---|---|
| 对象地址只剩 32 位 | 压缩的是存储表示，运行时会恢复为 64 位地址 |
| V8 只能使用 4GB 内存 | 4GB 约束的是压缩表示依赖的地址窗口 |
| 所有指针都可按 tagged value 解释 | tagged value 规则不适用于任意 C++ 指针字段 |

## 分析检查项

- 当前值是否为 tagged value
- 当前字段是否保存压缩指针或原始 C++ 指针
- `Smi` 与 `HeapObject` 标签位是否符合预期
- cage base / isolate root 是否参与地址恢复
