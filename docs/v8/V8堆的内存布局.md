# V8 堆的内存布局

## 概述

V8 Heap 按对象生命周期、对象大小和对象用途划分为多个空间。调试和漏洞分析中，堆空间决定对象是否容易搬移、何时被回收，以及地址观察是否稳定。

## 常用视角

以下结构为调试视角下的简化模型，不代表 V8 源码中的完整空间枚举。

```text
V8 Heap
├─ New Space
│  ├─ From-Space
│  └─ To-Space
├─ Old Space
├─ Large Object Spaces
│  ├─ Old Large Object Space
│  └─ Code Large Object Space
├─ Code Space
└─ Read-Only Space
```

## 空间职责

| 空间 | 主要内容 | 分析重点 |
|---|---|---|
| New Space | 新创建、短生命周期对象 | Minor GC、半空间复制、对象搬移 |
| Old Space | 存活时间较长的普通对象 | 晋升、地址稳定性、Major GC |
| Large Object Spaces | 大对象和大代码对象 | 通常不走半空间复制路径 |
| Code Space | JIT 代码和代码元数据 | 与普通 JS 对象分配路径不同 |
| Read-Only Space | 共享且不可变的内部对象 | 不按普通可变对象处理 |

## 对象流转

```text
new object
  -> New Space
      -> dead: Minor GC 回收
      -> survived: 晋升到 Old Space

large object
  -> Large Object Spaces

code object
  -> Code Space / Code Large Object Space
```

## Pointer Compression

Pointer Compression 影响 tagged value 的存储表示，不表示新的 heap space。它会改变字段中的值如何编码和解码，但不改变对象所属堆空间。

相关内容见：[V8 指针压缩机制](V8指针压缩机制.md)

## 分析检查项

- 目标对象位于 New Space、Old Space 还是 Large Object Space
- 目标对象是否可能在 GC 后搬移
- 目标对象是否为代码对象或只读对象
- 当前观察到的地址是否受到 pointer compression 影响
